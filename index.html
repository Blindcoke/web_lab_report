Предметне середовище: Предметним середовищем для лабораторної роботи є розробка веб-сайту для пекарні. Для цього використовується сучасний стек технологій, включаючи HTML, CSS і JavaScript для розробки фронтенду. Для управління версіями та спільної роботи над проєктом використовується система контролю версій Git та хмарний сервіс GitHub.

Тема: СТРУКТУРА HTML-ДОКУМЕНТА. ВИБІР ПРЕДМЕТНОЇ ГАЛУЗІ. РОБОТА З ПОСИЛАННЯМИ, ТАБЛИЦЯМИ, ЗОБРАЖЕННЯМИ, СПИСКАМИ в HTML-ДОКУМЕНТІ.

Мета: придбати практичні навички роботи з HTML-документом, таблицями, , зображеннями, посиланнями, списками, формами Створити шаблон звітного HTML-документом для відображення результатів роботи всіх лабораторних робіт.

Способи підключення стилів

Існує кілька способів підключення стилів до веб-сторінки: Вбудований CSS: Стилі можна вбудувати безпосередньо всередині HTML-документу, використовуючи тег <<style>> в розділі <<head>>. Зовнішній CSS: Стилі можна розмістити в окремому файлі CSS з розширенням .css, а потім підключити його до HTML-сторінки за допомогою тегу <<link>>. Inline CSS: Можна також задати стилі безпосередньо всередині тегів HTML за допомогою атрибуту style.

Зовнішній вигляд макету

За своїм розміщенням елементів і структурою, цей HTML-код схожий на "Блочний (Block)". У цьому випадку елементи такі як header, main, section, aside та footer використовуються для виділення різних частин сторінки, що розміщуються одна під одною або бічно. Кожна з цих частин сторінки має свою структуру та може включати додаткові вкладені блоки.

У цьому CSS-коді застосовується базова концепція блочного розміщення елементів на сторінці. Кожен елемент оформлення використовується для групування певних частин контенту сторінки. Найбільша увага приділяється використанню гнучких контейнерів (display: flex), що дозволяє створювати адаптивний і гнучкий дизайн.

Робота з флексбоксами дозволяє зручно та ефективно організовувати розміщення елементів на веб-сторінці.
Основні переваги роботи з флексбоксами включають:
Гнучкість: Флексбокси дають можливість зручно розміщувати елементи у веб-дизайні, незалежно від їхнього розміру та порядку. Простота використання: Синтаксис флексбоксів легко зрозуміти та використовувати, що робить їх дружніми до розробників.
Вертикальне та горизонтальне центрування: Флексбокси забезпечують зручні засоби для центрування елементів як по вертикалі, так і по горизонталі.
Адаптивність: Флексбокси дозволяють легко створювати адаптивний веб-дизайн, що забезпечує правильне відображення сторінки на різних пристроях та розмірах екрану.
Порядок: Зміна порядку елементів за допомогою флексбоксів дозволяє створювати різноманітні макети та покращує доступність контенту для користувачів.
Незважаючи на ці переваги, робота з флексбоксами має також деякі недоліки:
Сумісність: Незважаючи на те, що більшість сучасних браузерів підтримують флексбокси, можуть виникати проблеми з сумісністю на старих версіях деяких браузерів.
Складність: Деякі аспекти флексбоксів, такі як управління просторами, можуть бути складні для реалізації у великих проектах.
У цілому, флексбокси є потужним інструментом для створення адаптивних та зручних макетів у веб-дизайні, які забезпечують гнучкість та ефективність у роботі з розміщенням елементів.
